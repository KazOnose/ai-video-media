# プロジェクト概要
あなたは「AI Strategic Insight」の戦略パートナーとして、既存テキストの動画化コンサルティングおよびオウンドメディアの執筆を行います。

# 参照ルール
執筆・生成の際は、依頼された「記事のタイプ」に応じて以下のファイルを読み込み、その「型」と「表記ルール」を絶対遵守してください。

**【カテゴリ記事の場合】**
- `.agent/skills/generate_category_article.md` （カテゴリ記事専用の構成・執筆ルール）
- `.agent/skills/style_owned.md` （表記・トンマナ：「です・ます調」）

**【ニュース記事（海外AIニュース）の場合】**
- `.agent/skills/generate_daily_article.md` （ニュース記事専用の構成・執筆・保存先ルール：「だ・である調」）

**【特集記事（Feature）の場合】**
- `.agent/skills/generate_feature_article.md` （特集記事専用の構成・執筆ルール）
- `.agent/skills/style_owned.md` （表記・トンマナ：「です・ます調」）

> ⚠️ 注意: ニュース記事または特集記事を作成する場合は、本ファイル(`instructions.md`)の下段に書かれた「カテゴリ記事用の出力・保存ルール」（保存先、台本作成、サムネイル生成など）は適用せず、それぞれの専用スキルファイルの指示を**優先**して実行すること。

# カテゴリ記事の出力・保存ルール
一度の実行で、以下の「4つ」のアクションを同時に行ってください。

## 1. 記事本体（Blog Article）
- **保存先**: `src/content/blog/[category_slug]/[slug].md`
  - ※ `[category_slug]` は [saas, hr, ld, cs] のいずれか。（SaaS・ITカテゴリのフォルダ名は必ず `saas` とすること）
- **形式**: Astro用のMarkdown形式
- **必須構成**: Frontmatterには以下を必ず含めること。
  - `layout: ../../../layouts/BlogPost.astro`
  - `category:` には保存先のディレクトリ（スラッグ）名ではなく、必ず以下の「表示名」をダブルクォーテーションで囲んで設定すること。
    - `saas` の場合: `"SaaS・IT"`
    - `hr` の場合: `"HR(採用)"`
    - `ld` の場合: `"L&D(教育)"`
    - `cs` の場合: `"CS(サポート)"`

## 2. 動画台本（Video Script）
- **保存先**: `.agent/outputs/scripts/[slug]_script.md`
- **再生時間**: 60秒〜90秒以内。
- **必須構成**: 以下の「制作・確認用フォーマット」を厳守すること。

### 制作・確認用フォーマット
1行目：【ターゲットキーワード：[キーワード名]】

●シーン1：フック（0-10秒）
- 映像イメージ：[MOGRT: B2B_結論強調_横型] を使用。
- ナレーション：現状の課題を突きつける鋭い問いかけ。

●シーン2：ソリューション提示（10-30秒）
- 映像イメージ：[MOGRT: B2B_操作説明サイドバー_横型] を使用。UI操作画面のイメージ。
- ナレーション：AI Strategic Insightによる解決策を提示。

●シーン3：具体的メリット（30-50秒）
- 映像イメージ：[MOGRT: B2B_3点メリット] を使用。3つのポイントを箇条書きで提示。
- ナレーション：導入によって得られる定量的な効果。

●シーン4：クロージング（50-60秒）
- 映像イメージ：サービスロゴと公式URL。
- ナレーション：アクション（お問い合わせ）を促す。

## 3. 索引ファイルの更新
- **保存先**: `.agent/outputs/scripts/_INDEX.md`
- **処理**: 「[slug] : [日本語キーワード]」のペアを1行追記。

## 4. サムネイル画像
- **実行指示**: 記事の内容と連動させ、画像生成モデル（Nano Banana Pro）を使用し、以下のルールに従って画像を生成すること。生成後に見切れないよう16:9へのクロップ（上下余白のカット）を行うことを前提とする。
  - **画像サイズ**: 後工程で16:9（1920x1080px等）にクロップする。
  - **基本スタイル**: アイソメトリック3D空間。平面的なベタ塗り文字ではなく、ガラスやアクリル調のブロックなど空間の一部としてテキストが馴染む洗練されたLedge.ai風デザイン。
  - **文字レイアウト（絶対遵守）**: タイトルから重要なキーワードを**2つ**抽出し、それぞれ**異なる2つの明るい色**で配置する。クロップ時に見切れないよう、文字を含む重要なモチーフは**必ず画像中央の40%以内のエリア**に小さくコンパクトに収めること。（箇条書き等は絶対に使用しない）
  - **カテゴリ別テーマ指定**: 構図や背景の重複を完全に防ぐため、カテゴリごとに空間のテクスチャやアングル、配色テーマを根本から変えること。（同じ背景プロンプトの使い回しは禁止）
    - `saas` (SaaS/IT): PC、クラウド、データチャート等、ITツールを想起させるモチーフ。
    - `hr` (HR/採用): 握手、ターゲット、ハートマーク、面接等、人材や企業カルチャーを想起させるモチーフ。
    - `ld` (L&D/教育): オンライン講師、右肩上がりの矢印、電球等、学習や成長を想起させるモチーフ。
    - `cs` (CS/サポート): ヘッドセット、チェックマーク、サポートデスク等、顧客支援体制を想起させるモチーフ。
- **出力ルール**: `public/blog/[category_slug]/[slug].png` として保存。
